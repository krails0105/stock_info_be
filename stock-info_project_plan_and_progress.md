
# 📊 Stock-Info 프로젝트 기획서 & 진행 현황

## 1. 프로젝트 개요

**목표**  
주식 초보자를 위한 웹 서비스로,  
섹터 → 종목 → 상세 정보 흐름을 통해  
PER / PBR / 거래량 / 뉴스 등을 **쉽고 직관적으로 제공**하는 것이 목표.

**핵심 컨셉**
- 숫자 위주의 딱딱한 금융 사이트 ❌
- 처음 보는 사람도 흐름을 이해할 수 있는 주식 서비스 ⭕

---

## 2. 기술 스택

### Backend
- Spring Boot
- Spring MVC (Blocking 기반)
- REST API
- 외부 API 연동 (지수 / 종목 / 뉴스)

### Frontend
- React + Vite
- TypeScript
- SPA (CSR 기반)
- 향후 Next.js(SSR) 전환 가능성 고려한 구조

---

## 3. 전체 기능 로드맵

### MVP 핵심 흐름
```
홈 (지수 + 섹터)
 → 섹터 클릭
   → 종목 리스트
     → 종목 상세 (뉴스, 지표)
```

---

## 4. 백엔드 API 설계 및 구현 현황

### ✅ 구현 완료 API

| API | 설명 |
|----|-----|
| GET /api/indices | KOSPI / KOSDAQ 지수 |
| GET /api/sectors | 섹터 리스트 |
| GET /api/stocks | 섹터별 종목 조회 |
| GET /api/news | 종목별 뉴스 |

> 현재 모든 API는 조회(Read) 중심으로 구성되어 있음

---

## 5. 프론트엔드 구현 현황

### ✅ Step 1. 섹터 리스트
- 홈 화면에 섹터 카드 리스트 표시
- 섹터 클릭 가능

### ✅ Step 2. 종목 리스트 페이지
- 섹터 클릭 시 종목 테이블 표시
- 주요 컬럼: 종목명 / 현재가 / PER / PBR / 거래량
- 기본 정렬 기능 구현

### ✅ Step 3. 종목 상세 페이지
- 종목 기본 정보 표시
- 뉴스 리스트 연동
- 지수(KOSPI / KOSDAQ) 상단 표시

## 8. 기획 원칙 (중요)

1. **초보자 기준**
   - PER/PBR 설명 없이 숫자만 나열 ❌
   - “낮을수록 저평가” 같은 힌트 제공 ⭕

2. **기능은 단계적으로**
   - 처음부터 완벽한 금융 서비스 ❌
   - 작동 → 개선 → 확장 ⭕

3. **프론트 주도 UX**
   - 백엔드는 단순, 프론트에서 표현력 강화
